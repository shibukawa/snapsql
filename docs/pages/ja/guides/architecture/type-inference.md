# 型推論

このページでは SQL の戻り列やパラメータから型を推論する仕組みを説明します。

ポイント:

- DB カラム型 -> CEL 型 -> Go 型 のマッピングを行う
- NULL 許容性は SQL の情報とテンプレートのダミー値から決定される
- 配列や複合型はテンプレート内で明示的に扱えるように設計

推論アルゴリズムの要点:

- クエリの実行計画を使わずに静的解析で可能な限り型を確定する
- ambiguous な型はジェネレータに注釈を残し、手動で補正できるようにする

詳細は `docs/designdocs/20250913-query-gogen-ir-unification.ja.md` を参照してください。
